# Quick Compressor 開発・ビルド・配信手順 (.skill.md)

このファイルは、AIアシスタントが本プロジェクト「Quick Compressor」の開発、ビルド、およびリリース作業を安全かつ正確に進めるためのルールと指示書です。作業を開始する前に、必ずこのファイルの内容を確認し、以下の手順と設計思想に従ってください。

---

## 🚨 AI 共通ルール

1. **インストールの事前確認**:
   何かツールやライブラリをインストールする前に、必ず `where` コマンド（Windows環境では `where.exe`）を実行し、すでにインストールされていないか確認してください。
2. **日本語でのコミュニケーション**:
   ユーザーへの回答や説明は常に日本語で行ってください。
3. **初心者のユーザーへの配慮**:
   ユーザーが方針に迷っている場合や、要望が曖昧な場合は、必ず質問をして精度を向上させてください（最大30個まで）。また、可能な限り複数の選択肢を提示してください。

---

## 🛠️ プロジェクトの設計ルール

1. **実行ファイルのパス解決 (`sys.frozen`)**:
   - アプリケーションが PyInstaller によって .exe 化されている場合（`getattr(sys, 'frozen', False)` が真）、実行ファイルと同じ階層からファイルをロードするよう、動的にパスを解決してください。
2. **書き込み制限の回避 (%APPDATA%)**:
   - `C:\Program Files` などの保護されたディレクトリにインストールされた際、書き込みエラーを避けるため、ユーザー設定（`presets.json`）や動作ログ、アップデートキャッシュ（`config.json`）などは必ず `%APPDATA%\QuickCompressor` に保存してください。
3. **右クリックメニューの登録・解除**:
   - 右クリックメニューの登録・解除は `main.py` に引数 `--register` / `--unregister` を渡すことで行います。これは、Inno Setupインストーラーがインストール・アンインストール時に自動で呼べるようにするためです。
4. **自動アップデート検知とAPI制限対策**:
   - 起動時に非同期で GitHub Releases API (`https://api.github.com/repos/LunaFleuret/Quick-Compressor/releases/latest`) を確認します。
   - 同一IPアドレスからの過剰アクセスによるAPI制限を防ぐため、確認日時のタイムスタンプを `%APPDATA%\QuickCompressor\config.json` に保存し、**24時間に1回のみ**通信するようにキャッシュ処理を行ってください。
   - インターネット未接続時やAPI制限エラー(403)時は、エラーダイアログを出さずに静かに処理をスルー（サイレントスルー）してください。

## 🏗️ ビルドとインストーラー作成の手順

本アプリを他の人に配布可能な `setup.exe` にビルドする際は、必ず以下の手順で行ってください。

### 1. PyInstallerによる .exe 化
コマンドプロンプトやPowerShellで以下を実行、またはプロジェクト直下の `build.bat` を実行します。
```powershell
pyinstaller --noconsole --onefile --name "QuickCompressor" main.py
```
実行後、`dist\QuickCompressor.exe` が作成され、`default_presets.json` が `dist\` にコピーされます。

### 2. FFmpegおよびFFprobeの同梱
`dist\bin\` ディレクトリを作成し、Windows版の **`ffmpeg.exe`** と **`ffprobe.exe`** を手動でコピーして配置してください（※`ffplay.exe` は不要です）。
- 配置場所: `dist\bin\ffmpeg.exe`、`dist\bin\ffprobe.exe`

### 3. Inno Setupによるセットアップファイルの作成
Inno Setupのコマンドラインコンパイラ `ISCC.exe` を使用して `build_installer.iss` をコンパイルします。
```powershell
& "C:\Users\ユーザー名\AppData\Local\Programs\Inno Setup 6\ISCC.exe" build_installer.iss
```
（※Inno Setupのインストール先に応じて、`ISCC.exe` のパスを適宜解決してください。winget等でインストールされた場合は `HKCU:\Software\Microsoft\Windows\CurrentVersion\Uninstall\*` から検索できます。）

コンパイルに成功すると、`Output\QuickCompressor_Setup.exe` が生成されます。

## 📦 Git・GitHub 運用ルール

1. **不要なファイルの除外**:
   - 容量の大きいビルド生成物や、FFmpegバイナリなどは絶対に Git に含めないでください。
   - `.gitignore` に以下のディレクトリとファイルが正しく登録されていることを確認してください。
     - `build/`
     - `dist/`
     - `Output/`
     - `*.spec`
     - `user_presets.json`
2. **GitHub Releases での公開**:
   - リリースタグ名は必ず **`vX.X.X`**（例：`v1.0.0`）の形式にしてください（アプリの自動アップデート検知のパース処理と整合させるため）。
   - Assets エリアには、必ずビルドした **`QuickCompressor_Setup.exe`** を手動でアップロードしてください。
